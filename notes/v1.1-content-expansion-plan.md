## V1.1 scope (이 plan)

### 트랙 1: BGM 5 region (P0 retention killer 해결) — 3일 (Eng 결과 반영)

**목적:** 침묵 게임 → ambient audio. 첫 리뷰 / D1 retention / 전환율 보호.

**Architecture (autoplan Eng 권장):**
- **Audio singleton: `src/lib/audio.ts`** (zustand store 안 박음 — partialize 깨짐, non-serializable)
- **Wire 위치: `src/App.tsx`** (`useGame.subscribe(s => state.run.currentNodeId)` 로 region 변화 감지)
- **NOT in `src/components/RegionHeader.tsx`** (presentational component — layering mistake)
- **Cross-fade: HTMLAudioElement + WebAudio GainNode** (native-audio plugin 의 setVolume 보다 smooth)
- **Autoplay gesture gate: 첫 노드 tap 에서 `audioContext.resume()`** (WebView 기본 차단 우회)
- **Capacitor lifecycle: `App.addListener('appStateChange', ...)`** (전화/알림 시 pause/resume)

**범위:**
- 5 region 별 ambient BGM loop:
  - **forest-outskirts** — 안개 / 숲 결, 약하게 무게
  - **forgotten-mountains** — 차가운 바람 / 폐허 결, 짧은 박자
  - **ash-wastes** — 무감각 / 잿빛 결, drone heavy
  - **dreaming-city** — 너무 평화로운 결, 가짜 명랑함
  - **dawn-spire** — 차가운 새벽 빛 / 옛 마법사 결, choral 결
- 음원 출처 옵션 (가장 cost-effective 결정):
  - **A. CC0 사이트** (Pixabay Music, Free Music Archive, Incompetech) — 무료, selection 1일
  - **B. AI 생성** (Suno / Mubert / AIVA) — $10~30, custom prompt 로 region tone 정확히, 1일
  - **C. 한국 indie 음원 외주** — $50~150 per loop, 5일+ — V1.2 권장
- **포맷: Mono ogg-vorbis @ 64kbps** (Eng 권장 — mp3 아님). 목표 <800KB/loop, 총 <4MB.
- Capacitor 통합: HTMLAudioElement + WebAudio GainNode (cross-fade 30 줄, native-audio plugin 안 씀)
- Region 전환 시 cross-fade (0.5~1초). 로딩 중 전환 = load-queue 처리.
- 사용자 setting: BGM volume + on/off → `meta` 에 persist (zustand-persist 활용, 별도 localStorage 안 씀)

**작업량 추정 (Eng 정정):** 음원 1일 + audio singleton + cross-fade + lifecycle + autoplay gate + 디바이스 테스트 = **3일 CC**

**Risks (Eng 발견):**
- Bundle size: 5 mono ogg @ 64kbps = ~4MB (3MB → 7MB APK, 셀룰러 install 마찰 ~3x)
- WebView autoplay block: 첫 user gesture 전 차단 → autoplay gate 필수
- Background/lock/interrupt: Capacitor App lifecycle wire 안 하면 audio 끊김
- Cross-fade during loading: load-queue 없으면 에러

### 트랙 2: Echoes 축소 + lint-echoes 도구 — 4일 (Eng 결과 반영)

**🔴 Eng CRITICAL 발견:** [src/store.ts:252-268](src/store.ts#L252-L268) 의 `renderNodeText` 가 echoes 를 노드 entry 시 **한 번만** 평가. echo 의 knowledge 조건은 **STRICTLY 이전 노드에서 grant 된 것만 트리거**. lint:data 가 timing 안 잡음 → 22 echoes 중 3-5개 dead content 가능성.

**Sub-step 0 (선작성 필수): `scripts/lint-echoes.ts`** (1일)
- 각 echo 의 `condition.knowledge` 가 echo 의 노드 도달 전 어디선가 grant 되는지 reachability graph 검증
- `lint:data` 와 같은 walk pattern, 각 노드의 knowledge grant 시점 트래킹
- 도달 가능한 echo 만 통과, 나머지 error
- **22 echoes 작성 전** 도구 마련 → dead content 0%

**목적:** 첫 플레이어도 echoes 시스템이 있다는 결을 봄. meta-progression 만이 아닌 1회차 안에서도 echoes 트리거.

**범위:**
- **가지 노드 12개에 intra-run echo 1개 each = 12 echoes**
  - 조건: glimpse-of-mist / mist-meaning / templar-creed / rune-fundamentals / corruption-witnessed / forgotten-relic / shadow-banner / ash-prayer / buried-name / stranger-mark / *-respect / *-doubt
  - 1회차 안에 트리거 가능 (해당 knowledge 가 같은 회차에서 획득 가능한 노드 이전)
- **main path 의 핵심 노드 10개에 echo 1개 each = 10 echoes**
  - 보스 노드 (fo-boss-tree, fm-boss, fa-corrupted, dc-king, ds-mawang) + reveal 노드 (ds-throne, ds-floor-mages 등)
  - 이미 echoes 있는 노드 (fm-watcher, fa-watcher, dc-watcher, ds-watcher) 는 보강
- meta-progression echoes (chose-* / fled-*) = V1.2 또는 V1.1 reactive budget 으로 (사용자가 다회차 안 하면 작성해도 안 보임)

**작업량 추정 (Eng 정정):** lint-echoes.ts 작성 1일 + 22 echoes 작성 + re-lint 3일 = **4일 CC**

**합류 보장:** 기존 NodeEcho schema (condition + text) 활용. 신규 schema X.

## Risks / Open questions

- **R1:** BGM 음원 출처 (CC0 vs AI 생성 vs 외주) 가 결과 quality 에 큰 영향. AI 생성 시 region tone 정확도 검증 필수.
- **R2:** Bundle size 5-10MB 증가 — Play Store APK 100MB 이하 안전.
- **R3:** Capacitor audio plugin 의 background play / interrupt 동작 (전화 옴 / 알림 시) 검증 필요.
- **R4:** intra-run echoes 의 knowledge 조건이 실제로 same-run 안에 트리거 되는지 확인 (state store 의 knowledge 추가 시점 vs 노드 evaluate 시점).
- **Q1:** BGM 의 region cross-fade 가 노드 전환마다인가, region 전환 시만인가? (성능 고려)
- **Q2:** SFX (선택지 click, 보스 드롭) 도 V1.1 에 들어가나, BGM 만인가? — BGM 만이 cleaner.
