# chat-rpg 비주얼 스타일 가이드

이 문서는 chat-rpg의 모든 노드 일러스트가 따라야 하는 단일 source of truth. 새 이미지를 만들 때 (수동 / AI gen / commission 무관) 항상 이 문서와 대조해 일관성 검사한다.

스토리 바이블이 narrative source of truth라면, 이 문서는 visual source of truth.

## 1. 한 줄 결

**흑백 charcoal etching, 종이 결 살아 있는 사가 책 삽화.** 조선 후기 묵화 + 서양 동판화 + 1900년대 모험 소설 삽화의 교집합.

## 2. 매체 / 톤 / 결

### 매체
- **흑백 charcoal / pen etching / dry brush.** 디지털인데 종이 결을 잃지 않는다.
- 입자가 거칠다. 매끈한 그라데이션 / 디지털 블러 X.
- 종이 위에 잉크가 마른 느낌. 흰 자리는 종이 결, 검은 자리는 잉크의 두께.

### 톤
- **사가 / 옛이야기 / 책 삽화** 결. 게임 일러스트가 아니라 *오래된 책에서 떨어져 나온 한 페이지* 같은 인상.
- 정적. 시간이 멈춘 한 비트. 동작 중인 순간 X, 동작 직전 / 직후의 호흡 O.
- 멜랑콜리. 잔잔함 안에 무거움.

### 색
- **흑백 only.** 무채색. 회색 단계 5~7개로 충분.
- 약한 따뜻한 sepia tint 허용 (오래된 종이 결을 살리기 위해). 하지만 채도 5% 이하.
- 컬러 / 채도 / 강한 대비 절대 X.

## 3. 구도 룰

### 한 화면 한 비트
- 한 이미지 = 한 묘사 단위. 인물 + 배경 + 모티프 한 가지.
- 여러 사건 한 화면에 안 담음. 사가는 한 장에 한 사건.

### 인물 vs 풍경 비중
- **인물은 작게, 풍경은 광활하게.** 화면 면적의 80~90%가 풍경 / 분위기, 인물은 한 점.
- 이방인의 왜소함이 시각의 디폴트. 영웅으로 안 그린다.
- 단 회색의 자 / 마왕 / 보스 노드는 예외 — 인물 비중 50% 이상 가능.

### 시점
- **저각 / 멀리서 관찰**이 디폴트. 인물 어깨 너머 / 인물 위 / 인물 발 아래 정도.
- 캐릭터의 눈높이 close-up X (이방인은 익명. 얼굴 클로즈업 안 한다).
- 캐릭터 face shot은 회색의 자 / 마왕 / 보스에서만, 그것도 정면 X (실루엣 / 옆모습 / 그늘 진 얼굴).

### 빛
- **빛은 거의 없다.** 화면의 90%가 어둡거나 회색 톤.
- **중심 하이라이트 한 곳** — 안개 사이의 빛, 첨탑 끝의 빛, 거울 면, 룬의 빛 등. 한 화면에 한 빛만.
- 햇빛 / 강한 광원 X. 새벽 / 흐린 / 안개 너머의 빛이 디폴트.

### 비율 / 프레이밍
- 가로 4:3 또는 3:2 (모바일 portrait 안에서 위 1/3 차지하는 가로 패널).
- 이미지 가장자리는 부드럽게 페이드 / 종이 결로 사라짐 (hard edge X). 책 페이지에 인쇄된 결.

## 4. 핵심 모티프 시각 룰

### 안개
- 핵심 모티프. **거의 모든 화면에 안개**가 있다.
- 안개는 부드러운 그라데이션이 아니라 **거친 charcoal 텍스처 / 찢긴 종이 / 잿빛 입자**.
- 안개가 인물을 부분 가림. 완전히 가리거나 / 완전히 드러내거나 X — 항상 부분.

### 첨탑
- 항상 멀리. 안개 너머 잠깐 보이는 끝. 전체 첨탑을 한 화면에 안 담음.
- 첨탑은 검다 (마왕의 옥좌). 끝에 빛이 있을 수도 (옛 시대) / 없을 수도 (지금).

### 봉인
- 보지 않음. 첨탑 꼭대기까지 가서야 보이는 것이라 5지역 전엔 직접 안 그림.
- 단 그림자 / 사슬 / 살의 결로 암시 가능.

### 회색의 자
- **얼굴 안 보임.** 후드 / 안개 / 그늘로 가림. 1~4지역 절대 얼굴 안 보임.
- 회색 옷자락이 핵심. 등 / 옆모습 / 멀리서 본 모습.
- 5지역 reveal 후에야 윤곽 드러남 (그래도 face close-up X).

### 마왕
- 옥좌실 전엔 등장 X (혹은 실루엣 / 손 / 그림자만).
- 옥좌실에서 처음으로 형체 — 여전히 face shot은 그늘. 검은 갑옷 / 실루엣 / 옥좌의 무게가 인물보다 먼저 보인다.
- 거울 (3지역 fa-mirror) 에서 미리 한 번 — 이방인 자신의 거울 형태로.

### 이방인 (주인공)
- **익명.** 얼굴 안 그림. 등 / 어깨 너머 / 멀리서 본 실루엣.
- 어떤 인종 / 성별 / 연령으로도 안 잡힘. 망토 / 후드 / 흐릿한 윤곽.
- 검 또는 양피지를 든 손은 가능. 얼굴은 X.

### 동료들
- 아스트리드 / 베일런 등은 face profile 가능 (정면 X). 옆모습 / 어두운 자리에서 옆얼굴.
- 실제 portrait은 별도 `public/portraits/`에 있고 일러스트는 풀 비주얼 톤.

### 부패 / 마왕 결
- **검은 액체 / 검은 뿌리 / 결은 흙 깊이로 흐름**. 빨간 색 X (모든 색 금지).
- 단단한 유기체 형태가 아니라 *침투하는 결*. 흙에 스며들고 돌에 박히고.

## 5. 부정 prompts (NG 리스트)

다음 결은 절대 안 나옴:
- 색채 / 채도 / 강한 대비
- Photorealism / hyperrealism
- Anime / 만화체 / chibi / Disney / Pixar
- 3D 렌더링 / CGI 결
- 현대 요소 (자동차, 전화, 도시 야경, 네온, 스마트폰 등)
- 캐릭터 face close-up (특히 이방인 / 회색의 자 / 마왕)
- 영웅 포즈 (검을 들어올린 자세, 결연한 표정 등)
- 강한 광원 / 햇빛 / 푸른 하늘
- 환한 배경 / 깨끗한 라인
- 디지털 블러 / 매끈한 그라데이션
- 텍스트 / 글자 / 자막 (이미지 안에 글자 X)
- 워터마크 / 사인
- 군중 / 여러 인물 (한 화면 한 인물 디폴트)
- 동물 일러스트 결 (썩은 늑대 / 부패 나무 등 보스도 *자연스러운 부패의 결*로, 디즈니 동물 X)

## 6. 기술 스펙

### 출력 포맷
- **포맷**: webp (region SVG와 다름 — 사진성 일러스트는 webp)
- **해상도**: 1280×720 (가로 16:9) 또는 1280×960 (4:3) — 모바일 portrait 안에서 위 1/3 차지
- **파일 크기**: 200~400KB 권장. 600KB 초과 금지
- **이름**: `<node-id>.webp` (예: `p-genesis.webp`, `fo-arrive.webp`)
- **위치**: `public/scenes/<region>/<node-id>.webp` (region 폴더 안에)

### 뱅크 / fallback
- 노드 image 필드가 비어 있으면 region.headerImage 로 fallback (기존 SVG)
- 그래서 이미지가 점진적으로 추가돼도 깨지지 않는다

## 7. AI gen prompt 형식

image-artist agent가 따르는 prompt 템플릿:

```
[Subject + action]: <노드의 한 비트를 한 줄로>
[Composition]: <인물 위치 / 풍경 비중 / 시점>
[Style]: black and white charcoal etching, paper texture, dry brush ink, 1900s adventure novel illustration
[Mood]: melancholic, still, saga, quiet weight
[Lighting]: minimal, single soft highlight in <위치>
[Negative]: color, photorealism, anime, modern elements, character close-up, heroic pose, bright lighting, text in image
[Aspect]: 4:3 horizontal
[Reference]: 서울 2033 / Frank Pape illustrations / 김홍도 묵화 / 19th century adventure novel illustrations
```

각 prompt는 **노드 description / region context / persona MDs**를 읽고 그 노드의 핵심 비트 하나만 시각화한다. 노드의 모든 비트를 다 담으려고 하지 않는다.

## 8. 참조 이미지

(사용자가 첨부한 서울 2033 스크린샷 또는 외부 reference URL을 여기 적음.
현재 conversation에서 받은 서울 2033 결 — 흑백 etched / 종이 결 / 안개 + 첨탑 + 사람 작은 점 — 이 디폴트 reference.)

## 9. 일관성 점검

새 이미지가 기존 갤러리와 일관되는지 빠른 점검:
- 채도 0% / 흑백? ✓
- 종이 결 / charcoal 결 살아 있나? ✓
- 안개 들어갔나? (없으면 의도 명시) ✓
- 인물 face close-up 아닌가? ✓
- 영웅 포즈 / 강한 광원 / 만화체 아닌가? ✓
- 한 화면 한 비트인가? ✓
- 파일 크기 600KB 이하인가? ✓
- 파일 이름 / 위치 컨벤션 맞나? ✓

8개 모두 ✓ 면 갤러리에 추가. 하나라도 ✗면 재작업.

## 10. 진화 / 변경

이 문서는 visual canon. 변경하려면:
1. 사용자 명시 결정 (스타일 자체를 바꿀 때)
2. 또는 작은 보강 (모티프 추가 / NG 추가) — 이건 점진적 추가 OK

기존 갤러리를 retcon하는 변경은 신중. 한 번 결정한 결을 바꾸면 모든 이미지 재작업이 따라옴.
